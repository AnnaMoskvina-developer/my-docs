# Добро пожаловать в демонстрационный проект технического писателя

Этот проект создан для позиции **технического писателя в VK Tech**.  
Здесь я показываю, как применяю подход **Docs as Code** на практике.

- **Git** — контроль версий документации
- **Markdown** — написание текстов (таблицы, сноски, предупреждения, изображения)
- **MkDocs** — сборка статического сайта
- **GitHub Actions** — автоматическая публикация при каждом обновлении
- **GitHub Pages** — хостинг готовой документации

## ⚙️ Как это устроено

- Весь код документации хранится в **Git**.
- При каждом `git push` **GitHub Actions** автоматически собирает сайт через `mkdocs build` и публикует его в ветку `gh-pages`.
- **GitHub Pages** отдаёт готовый сайт по публичной ссылке.

Это полностью воспроизводит реальный процесс работы технического писателя в команде, где используется Docs as Code.
